# To Site rubric

A passing site is a concise evidence reader, not a generic documentation shell.

## Semantic

- Landing page states the thesis, evidence-to-decision path, and proof limit before deep detail.
- Every public claim keeps its qualifier, evidence status, source refs, and proof limit.
- Diagram arrows, nodes, and trust boundaries map to source-backed model objects.
- `does_not_prove` and unknown states are legible rather than hidden in a footer.

## Visual direction

- One page job and one dominant subject-grounded visual model per page.
- Typography, dividers, labels, and number rails encode real information rather than decoration.
- A restrained signature is permitted; a generic dashboard/card grid is not.
- Desktop and mobile have readable, purpose-designed diagram layouts; mobile is not a shrunken desktop SVG.

## Accessibility and delivery

- Keyboard focus, headings, landmarks, descriptive links, text equivalents, contrast, and non-color status cues pass inspection.
- `prefers-reduced-motion` disables nonessential motion.
- Static links and deep routes work under the configured base path.
- `dist/`, semantic map, validation reports, desktop/mobile screenshots, hashes, and external revision pins are in the review trail.
