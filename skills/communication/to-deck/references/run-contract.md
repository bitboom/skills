# Run contract

Never overwrite a prior round. Use two-digit versions beginning with v01.

    <run>/
    ├── 00-deck-brief.yaml
    ├── 00-brief-decision.md
    ├── input/
    │   ├── point.md
    │   ├── point.yaml
    │   └── point.sha256
    ├── slides/
    │   ├── v01-storyboard.md
    │   ├── v01-visual-model.yaml
    │   ├── v01-deck.pptx
    │   ├── v01-render/
    │   ├── v01-diagram-review.json
    │   ├── v01-reader-review.json
    │   ├── v01-executive-review.json
    │   ├── v01-visual-review.json
    │   ├── v01-baseline-review.json
    │   ├── v01-gate.json
    │   └── v01-decision.md
    ├── final/
    │   ├── final-deck.pptx
    │   └── final-report.md
    ├── manifest.json
    └── review-trail.zip

## Review identity

Every review JSON contains round, point_sha256, critical_issues, and issues. The gate rejects a review written against another Point.

Diagram review records structural_elements, relationships, diagram_required, dominant_visual_present, semantic_visual_share, node/edge/boundary totals and checked counts, and diagram direction errors.

Reader review records teach_back_match, headline_paraphrase_match, identified_input, identified_decision_maker, identified_action, identified_outcome, and critical_questions.

Baseline review is mandatory when an earlier deck exists. It records no_regression booleans for mental_model_reconstruction, scanability, and information_coverage, plus regressions.

Gate JSON is generated by pipeline.py and must not be hand-edited.
