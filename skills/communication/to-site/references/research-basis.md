# To Site research basis

- W3C Web Annotation Data Model: claim evidence should retain a useful target/locator, not only a homepage URL. https://www.w3.org/TR/annotation-model/
- W3C PROV-O: record Point input → projection → build → public artifacts as derivation/provenance. https://www.w3.org/TR/prov-o/
- Shneiderman, overview first, zoom/filter, details on demand: use landing thesis before evidence depth. https://doi.org/10.1109/VL.1996.545307
- Larkin & Simon: reserve diagrams for relations where spatial layout reduces inference. https://doi.org/10.1111/j.1551-6708.1987.tb00863.x
- Ginns: place visual explanations adjacent to the referenced visual element. https://doi.org/10.1016/j.learninstruc.2006.10.001
- WCAG 2.2: non-text alternatives, structure, keyboard access, and non-color semantics. https://www.w3.org/TR/WCAG22/
- Reproducible Builds: record inputs, pinned dependencies, commands, and output hashes. https://reproducible-builds.org/docs/definition/
- Archify (`tt-a1i/archify`, pin `ed0efcc763d358b78df845182b5ed24a9d165a1c`): optional typed technical-diagram renderer; retain its IR and validation but keep semantic gates separate. https://github.com/tt-a1i/archify
- Anthropic `frontend-design` (pin `fa0fa64bdc967915dc8399e803be67759e1e62b8`): use its subject-specific visual direction and screenshot critique as a reference only. https://github.com/anthropics/skills/tree/main/skills/frontend-design
