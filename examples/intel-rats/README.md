# Intel RATS example

This example applies `point` before `to-deck` to `bitboom/Intel-rats`.

- [`brief.yaml`](brief.yaml) records the Point communication contract.
- [`point-result.md`](point-result.md) records the domain-first passing content model and gate evidence.
- [`pilot-result.md`](pilot-result.md) preserves the rejected first deck as regression evidence.
- [`to-deck-result.md`](to-deck-result.md) records the accepted Visual Component Selection and slide-gate result.
- [`final-deck.pptx`](final-deck.pptx) is the accepted one-slide Intel TDX remote-attestation explainer.
- [`review-trail.zip`](review-trail.zip) contains the immutable Point, baseline, A/B component artifacts, reviews, hashes, render, and final report.

The run compares a project-blind model prior and authoritative domain baseline with a baseline-blind source map before drafting. After Point approval, To Deck extracts visual jobs, independently scores two component ensembles, selects a causal pipeline, renders the slide, and runs isolated semantic, novice, executive, writer, visual, and regression reviews.
